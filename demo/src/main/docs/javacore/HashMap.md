HashMap
1.JDK7实现原理
    HashMap的实现使用了一个数组，每个数组项里面有一个链表的方式来实现；
    因为HashMap使用key的hashCode来寻找存储位置；
    不同的key可能具有相同的hashCode，这时候就出现哈希冲突了；
    为了解决哈希冲突，有开放地址方法，以及链地址方法；
    HashMap的实现上选取了链地址方法；
    也就是将哈希值一样的entry保存在同一个数组项里面；
    可以把一个数组项当做一个桶，桶里面装的entry的key的hashCode是一样的。

2.JDK8实现原理
    在JDK7种，如果一个链表的长度过长，那么查找的时间复杂度是O(n)
    也就是链表多长，就需要多少次查询
    
    JDK8中为了提高查询效率，当链表长度大于8时，将重新优化为红黑树
    时间复杂度降为O(logn)
    
3.为什么说HashMap不是线程安全的
    a) 如果2个线程同时put数据，又恰巧根据key hashcode出来的地址是相同的
       那么就会产生数据与预期不符的情况
    b) resize导致的循环链表的问题
